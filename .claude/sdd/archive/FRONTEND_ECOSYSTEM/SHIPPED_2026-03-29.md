## Lições Aprendidas

### Processo

- **Subagents nao herdam permissao de Write.** Todos os 12 subagents delegados foram bloqueados. Para builds grandes, criar arquivos diretamente no agente principal — nunca delegar criação de arquivos a subagents.
- **62 arquivos e demais para uma sessao.** Dividir em 2 sessoes: Build A (agentes + integrações) e Build B (conteudo KB) funcionou via `/continuar`.
- **O `/continuar` funciona.** Gap analysis automatico + execução apenas do que falta, sem reescrever o que ja existe. Pattern validado.

### Técnico

- **Paralelo exato DE→Frontend funciona.** A estrutura de KB (index + quick-ref + concepts + patterns) e o template de agentes (T2, KB-First, Agreement Matrix) se aplicam perfeitamente a um dominio nao-DE. Prova de que o AgentSpec e extensivel alem de data engineering.
- **KB structure (index + quick-ref) e suficiente como MVP.** Agentes podem consultar MCP context7 como fallback enquanto concepts/patterns sao completados.

### Comunicação

- **Brainstorm informal → DEFINE direto funcionou.** As notas detalhadas do usuario tornaram a Fase 0 formal desnecessaria. Clarity Score 14/15 sem brainstorm formal.
- **Perguntas do /define clarificaram escopo.** Stack (React+Next+Tailwind), fases do UX designer, e decisão sobre FRONTEND_CONVENTIONS foram resolvidos com 3 perguntas.

### Ferramentas e Bibliotecas

- **`_template.md` existente e robusto.** Criar 5 agentes T2 seguindo o template foi mecanico — zero decisao de estrutura.
- **KB templates (`index.md.template`, `concept.md.template`, `pattern.md.template`) padronizam a criacao.** Conteudo varia, estrutura nao.
- **MCP context7** funciona para React/Next.js/Tailwind docs — zero MCP novo necessario.
